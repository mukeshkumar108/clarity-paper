Build a polished full-stack SaaS web app called “Clarity”.

Clarity helps ordinary people understand important documents before they sign, reply, or make a decision. Users can upload or paste a document, receive a plain-English summary, see key clauses, risks, obligations, deadlines, confusing terms, and ask questions about the document.

Important:
Clarity is NOT a law firm.
Clarity does NOT provide legal advice.
Clarity explains documents in plain English and helps users identify issues they may want to discuss with a qualified professional.

Target users:
- renters reviewing tenancy agreements
- employees reviewing employment contracts
- freelancers reviewing client contracts
- small business owners reviewing supplier/client terms
- people receiving legal letters
- people reviewing NDAs, service agreements, invoices, policies, or settlement documents

Document analysis prompt:
The LLM should return strict JSON:
{
  "briefSummary": string,
  "plainEnglishSummary": string,
  "documentType": string,
  "keyPoints": [
    {
      "title": string,
      "explanation": string,
      "importance": "low" | "medium" | "high"
    }
  ],
  "keyClauses": [
    {
      "clauseName": string,
      "plainEnglishMeaning": string,
      "whyItMatters": string,
      "riskLevel": "low" | "medium" | "high",
      "sourceText": string | null
    }
  ],
  "obligations": [
    {
      "who": "user" | "other_party" | "both" | "unknown",
      "obligation": string,
      "deadline": string | null,
      "riskIfMissed": string | null
    }
  ],
  "deadlines": [
    {
      "dateOrTiming": string,
      "description": string,
      "importance": "low" | "medium" | "high"
    }
  ],
  "risks": [
    {
      "title": string,
      "explanation": string,
      "riskLevel": "low" | "medium" | "high",
      "whatToCheck": string
    }
  ],
  "unusualTerms": [
    {
      "term": string,
      "whyItMayBeUnusual": string,
      "whatToAsk": string
    }
  ],
  "missingInfo": [
    {
      "item": string,
      "whyItMatters": string
    }
  ],
  "questionsToAsk": [
    {
      "question": string,
      "whyAskThis": string
    }
  ],
  "confidenceNotes": string
}

Legal safety rules:
- Never say “this is legally valid”
- Never say “you should sign”
- Never say “you should sue”
- Never give definitive legal advice
- Use wording like:
  - “This may mean…”
  - “You may want to check…”
  - “This could be worth asking…”
  - “A qualified professional can confirm…”
- Always include disclaimer:
  “Clarity provides document explanation, not legal advice.”

Document Q&A:
User can ask questions about the uploaded document.
Examples:
- “What am I agreeing to?”
- “Can they terminate this easily?”
- “What happens if I cancel?”
- “Are there any fees?”
- “What should I ask before signing?”
- “Explain clause 8 in simple terms.”

Q&A rules:
- Answer only from the document text
- If unsure, say so
- Do not invent clauses
- Do not provide legal advice
- Suggest checking with a professional for high-risk issues
- Keep answers plain-English

Limit free users to 2 questions per document.
Pro users get more.
